# APGAP Influenza Pipeline

A Nextflow pipeline for influenza whole-genome analysis using **IRMA**, **BLAST**, and **VADR**.

The pipeline performs:

- Read quality control
- Genome assembly
- Coverage assessment
- Segment identification
- Genome annotation
- Amino acid variant annotation

from paired-end Illumina sequencing data.

---

# Workflow

```text
FASTQ Files
    │
    ▼
(Optional) Primer Trimming
    │
    ▼
Quality Control
(FastQC → fastp → FastQC → MultiQC)
    │
    ▼
IRMA Genome Assembly
    │
    ▼
Coverage Assessment
    ├── Coverage Report
    ├── PASS Segment List
    └── PASS Segment FASTA
    │
    ▼
BLAST
(Segment Identification)
    │
    ▼
VADR Annotation
(CDS and Gene Annotation)
    │
    ▼
Variant Annotation
(Map IRMA variants to VADR annotations)
    │
    ▼
Final Results
    ├── Consensus Sequences
    ├── Coverage Statistics
    ├── Segment Identification
    ├── Gene/CDS Features
    ├── Amino Acid Variants
    └── Mutation Effects
```

# Requirements

- Nextflow (DSL2)
- Java 17 or later
- Apptainer/Singularity
- R (for coverage and variant annotation)

# Input

The pipeline accepts paired-end Illumina FASTQ files.

Example directory structure:

```text
data/
├── Sample1_R1.fastq.gz
├── Sample1_R2.fastq.gz
├── Sample2_R1.fastq.gz
└── Sample2_R2.fastq.gz
```

---

# Running the Pipeline

```bash
nextflow run main.nf \
    -c nextflow.config \
    --reads_dir data \
    --outdir results \
    -resume
```

---

# Running on a SLURM Cluster

Example submission script (`run_pipeline.sh`):

```bash
#!/bin/bash
#SBATCH -p htc
#SBATCH --mem=100G
#SBATCH --time=4:00:00
#SBATCH --cpus-per-task=4
#SBATCH -o jobout/pipeline_%j.out
#SBATCH -e jobout/pipeline_%j.err
#SBATCH --mail-type=ALL

module load openjdk
module load nextflow

export APPTAINER_CACHEDIR=$HOME/.apptainer/cache
mkdir -p "$APPTAINER_CACHEDIR"

export NXF_OPTS='-Xms1g -Xmx8g'

nextflow run main.nf \
    -c nextflow.config \
    --reads_dir data \
    --outdir results \
    --threads ${SLURM_CPUS_PER_TASK} \
    -resume
```

Submit the job:

```bash
sbatch run_pipeline.sh
```

# Output

The pipeline creates the following directory structure:

```text
results/
├── qc/
├── trim/
├── assembly/
├── coverage/
├── blast/
├── vadr/
└── variant_annotation/
```

## Output Directories

| Directory | Description |
|-----------|-------------|
| `qc/` | FastQC and MultiQC reports |
| `trim/` | Trimmed FASTQ files (optional) |
| `assembly/` | IRMA assembly outputs and consensus sequences |
| `coverage/` | Coverage statistics and PASS segment FASTA |
| `blast/` | BLAST results for segment identification |
| `vadr/` | Gene and CDS annotations generated by VADR |
| `variant_annotation/` | Amino acid variant annotations and mutation effects |

---

# Coverage Assessment

Coverage is calculated for each influenza genome segment.

Segments with a median coverage greater than or equal to the specified threshold (default **20×**) are classified as **PASS**.

Generated files include:

- `sample_coverage.tsv`
- `sample_pass_segments.txt`
- `sample.pass_segments.fa`

Only PASS segments are used for downstream annotation.

---

# Variant Annotation

The pipeline combines:

- IRMA nucleotide variants
- BLAST segment identification
- VADR gene/CDS annotations

to report:

- Gene name
- Codon number
- Codon position
- Reference amino acid
- Alternate amino acid
- Amino acid change
- Synonymous or nonsynonymous mutation
